2. 1990년대 후반 ~ 2000년대
1997년, 에릭 레이먼드는 해커 커뮤니티와 자유 소프트웨어 원칙에 대한 분석인
<대성당과 바자>를 발표하고 1998년 초에 주목을 받으며 Netscape Communications Corporation이
인기있는 Netscape Communicator 인터넷 제품군을 무료 소프트웨어로 출시하도록
동기를 부여한 요소였고 이것이 모질라 파이어 폭스와 모질라 썬더 버드의 기초가 됩니다.
그리고 그는 리눅스 대회 이전 2개의 무료 소프트웨어 개발 모델에 이어
지배적인 원칙을 재확인한 뒤, 좋은 오픈 소스 소프트웨어를 만드는 데 필요한 19가지 지침을
설명하고 소스 코드 공개에 대한 최종결정을 내렸습니다. 그러면서 넷스케이프가
최첨단 웹 브라우저를 위해 소스 코드를 최종 발표한 1998년 많은 사람들이 버그를 찾아
수정할 수 있게 함으로써 개선되고 Netscape Release를 둘러싼 관심은 열린 개발 프로세스의
우수성을 교육, 옹호할 수 있는 기회를 창출해냅니다. 그리고 당시 대성당 모델처럼
개발이 중앙 집중식으로 이루어졌던 전통적 개발 모델과 비교해 바자 모델은
역할이 명확히 나누어져있고 바자 모델을 사용해 개발한 소프트웨어는 다음의 특성을 지닙니다.
1. 사용자는 공동 개발자로 취급한다.
- 사용자는 공동 개발자처럼 취급되므로 소프트웨어의 소스 코드 액세스가 가능해야 하고
소프트웨어 추가, 코드 수정, 버그 보고서, 문서 제출 등의 방법으로 개발자의 역할을 합니다.
기존의 모델처럼 개발자가 단독으로 위의 과정을 수행하면 소프트웨어에서 발생하는 버그를
찾는 데에 한계에 부딪치게 되어 일정 수준 이상의 문제를 해결하는 데에 큰 어려움이 생기지만
여러 사용자가 함께 문제에 접근하면 리누스 법칙의 "눈알이 많으면 모든 버그가 얕다."
라는 얘기처럼 많은 사용자가 해당 소프트웨어 소스 코드를 보며 개발자 단독으로
진행했던 데에 비해 더 많고 폭넓은 버그를 찾아 수정할 수 있고 해결 방법을 제안할 수 있습니다.
즉, 개발자가 새로 만든 소프트웨어를 베타 테스트와 같은 버전으로 사용자들에게
소스 코드와 함께 그 프로그램을 공개하면 사용자는 자기만의 테스트 환경에서
그 프로그램을 사용하면서 불편한 점이나 개선이 필요한 점들을 문서로 개발자에게 보내고
고급 프로그래밍 기술을 소유하고 있는 사용자라면 직접 소프트웨어의 소스 코드에 접근하여
문제가 되는 코드를 찾아 수정하고 해당 코드에 대한 설명이 담긴 주석을 달아줌으로써
다른 사용자들이 알아보고 이해할 수 있도록 하는 것입니다. 이렇게 수정된 소프트웨어는
처음의 버전과 비교했을 때 보안상 취약점이나 기능 개선 등의 효과가 있음을 확인할 수 있습니다.
2. 소프트웨어 조기 배포
- 소프트웨어의 첫 버전은 조기에 공동 개발자를 찾을 수 있도록 가능한 한 빨리 배포해야 합니다.
